Environment Variables

CountElysia uses environment variables for configuration. Below are all available environment variables and their descriptions.

Application Configuration

Variable Default Description
APP_NAME CountElysia Application name
APP_PORT 3000 Application listening port

Database Configuration

Variable Default Description
DB_USER - Database username
DB_PASSWORD - Database password
DB_HOST localhost Database host address
DB_PORT 3306 Database port
DB_NAME - Database name

Rate Limit Configuration

Variable Default Description
GET_RATE_LIMIT_MAX 10 GET request rate limit: max requests per time window
POST_RATE_LIMIT_MAX 1 POST request rate limit: max requests per time window
GET_RATE_LIMIT_DURATION 60000 GET request rate limit: time window duration (ms)
POST_RATE_LIMIT_DURATION 60000 POST request rate limit: time window duration (ms)

Tip: Ensure strong passwords are used in production environments and restrict database access permissions. The .env file should not be committed to version control.

Base URL

The base URL for all API requests is as follows:

Base URL 
http://localhost:3000/api

Get Counter

Retrieve the current value of a counter by name. If the counter doesn't exist, it will be automatically created with an initial value of 0.

GET /api/count/:name Get Counter

This endpoint retrieves the current value of a counter by name. If the counter doesn't exist, the system automatically creates it and initializes it to 0.

Path Parameters

Parameter Type Required Description
name string Yes Unique identifier name for the counter

Request Example

Request 
curl -X GET http://localhost:3000/api/count/my-counter

Response Example

Response (200 OK) 
{
  "success": true,
  "message": "Query success",
  "data": {
    "name": "my-counter",
    "count": 5
  }
}

Increment Counter

						
Increment the counter by name by 1 and return the new count value. If the counter doesn't exist, it will be automatically created and initialized to 1.


						

							

								POST
								/api/count/:name
								Increment Counter
							

							

								

									
This endpoint increments the counter by name by 1. This is an idempotent operation - calling it multiple times will keep increasing the counter value.
